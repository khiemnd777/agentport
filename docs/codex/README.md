# Codex Notes

These are project documentation files for humans and Codex. They are not Codex configuration files.

Official Codex project instructions live in `AGENTS.md`.

Use `.codex/config.toml` only when this project needs official project-scoped Codex configuration. Do not place arbitrary notes under `.codex/`.

- `techstack.md`: stack, architecture, commands, ports, config.
- `memory.md`: durable product and implementation decisions.
- `runbook.md`: common local run, validation, and recovery steps.
